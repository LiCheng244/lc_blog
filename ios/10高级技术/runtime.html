关于 runtime 的使用

引言:

  1. 是一种面向对象编程语言的运行环境，类似 java 的虚拟机。
  2. C语言，函数的调用在编译的时候会决定调用哪个函数。
  3. OC 在编译的时候并不能决定真正调用哪个函数，只有在真正运行的时候才会根据函数的名称找到对应的函数来调用。
  4. 程序启动时首先加载运行时（start），它是 OC 的核心，OC 的底层， OC是基于运行时的。
  5. 运行的时候将 oc代码转换成runtime方式， 通过runtime底层的消息机制来运行， 将结果反馈到页面上来展示.

# I. 发送消息

# 用途：

  1. 方法调用的本质，就是让对象发送消息。
  2. objc_msgSend,只有对象才能发送消息，因此以 objc 开头。
  3. 使用消息机制，必须导入#import <objc/message.h>

# 发送消息：

// 本质：让对象发送消息
objc_msgSend(p, @selector(eat));

// 用类名调用类方法，底层会自动把类名转换成类对象调用
objc_msgSend([Person class], @selector(eat));

# II. 给分类添加属性

# 用途：

最多的用在分类中：

  • 给分类动态添加属性，分类默认是不允许添加属性的， 能做到更好的解耦, 简化使用。
  • 给一个类声明属性，其实本质就是给这个类添加关联，并不是直接把这个值的内存空间添加到内存空间。
  • SDWebImage 分类里用到过。

# 方法：

1.利用关联对象 获取属性：

在调用运行时之前 从 关联对象 中获取对象属性，判断是否已经取到属性，如果取到直接返回。

- (NSString *)name
{
// 根据关联的key，获取关联的值
return objc_getAssociatedObject(self, key);
// 参数1: 对象， 参数2: 动态属性的 key
}

2. 利用关联对象，将获取到的属性动态添加：

// 利用关联对象 动态添加属性
- (void)setName:(NSString *)name
{
objc_setAssociatedObject(self, key, name, OBJC_ASSOCIATION_RETAIN_NONATOMIC);
// 参数1: 给哪个对象添加关联 self
// 参数2: 动态添加属性的 key, 获取值的时候使用
// 参数3: 动态添加的属性值
// 参数4: 对象的引用关系
}

# III. 动态获取类的属性

# 用途：

  1. 将获取到的属性，可以使用 kvc 来设置属性值。
  2. 运行时 字典转模型，设置属性值。
  3. 所有的字典转模型第三方框架的核心算法（通过运行时动态获取类的属性，通过 kvc 来设置属性值）！
  4. 基本上都是这个思路， 但是 会复杂 比如 字典的嵌套等等。

# 方法：

// 声明
/**
获取类的属性列表

@return 属性列表数组
*/
+ (NSArray *)lc_objProperties;



// 实现
const char * kPropertiesListKey = "Properties";
+(NSArray *)lc_objProperties {

// 0. 从‘关联对象’中获取对象属性，如果有值，直接返回
/**
方法: objc_getAssociatedObject： 获取关联对象 动态添加的属性

参数:
1. 对象
2. 动态属性的 key

返回值:
- 动态添加的属性列表
*/
NSArray *ptyList = objc_getAssociatedObject(self, kPropertiesListKey);
if (ptyList != nil) {
return ptyList;
}


// 1. 调用运行时方法，获取类的属性
/**
方法: class_copyXXXXList

- Ivar : 成员变量
- Method : 方法
- Property : 属性
- Protocol : 协议

参数:
- 要获取的类
- 类属性的个数指针，用来保存 属性个数

返回值:
- 返回所有属性组成的数组, C语言中，数组的名称就是指向数组第一个元素的地址
*/
unsigned int count = 0;
objc_property_t *proList = class_copyPropertyList([self class], &count);


// 2. 遍历数组，获取属性名
NSMutableArray *properties = [NSMutableArray array];
for (unsigned int i = 0; i < count; i++) {

// 1> 从数组中取出属性
objc_property_t property = proList[i];

// 2> 从属性中取出属性名称 c语言字符串
const char *cName = property_getName(property);

// 3> C字符串 转 OC 字符串
NSString *name = [NSString stringWithCString:cName encoding:(NSUTF8StringEncoding)];

// 4> 添加到数组中
[properties addObject:name];
}


// 3. 释放数组
free(proList);


// 4. 利用关联对象 动态添加属性
/**
objc_setAssociatedObject ：利用关联对象 动态添加属性

参数
1. 对象 self
2. 动态添加属性的 key, 获取值的时候使用
3. 动态添加的属性值
4. 对象的引用关系
*/
objc_setAssociatedObject(self, kPropertiesListKey, properties.copy, OBJC_ASSOCIATION_RETAIN_NONATOMIC);

// 5. 返回数组
return properties.copy;
}


// 调用
// 获取 person 类的属性列表
NSArray *properties = [Person lc_objProperties];
NSLog(@"%@", properties);

# IV. 交叉方法（黑魔法）Method Swizzling

# 用途：

  • 对系统方法或者第三方框架的方法不满意时，我们希望自己写一个方法。
  • 利用交换方法，先执行 自己的方法 再执行系统或者第三方框架的方法。
  • AFN：交换了NSURLSession 的 resume 和 supsend两个系统方法， 当网络请求开始、网络请求挂起时， 发出通知，监听网络状态变化。
  • 记住： 在迫不得已的时候再使用交换方法，否则不要使用。

# 方法：

  • 获取交换的实例方法： class_getInstanceMethod([self class], @selector(viewWillAppear:));
  • 交换方法：method_exchangeImplementations(originalMethod, swizzledMethod);
#import <objc/runtime.h>

@implementation UIViewController (Tracking)

// 在每个类被装载到Runtime的时候调用该方法
// 因为方法交叉影响的是全局状态，+load中能保证在class 装载的时候进行交叉
+ (void)load {

// 保证只交叉一次
static dispatch_once_t onceToken;
dispatch_once(&onceToken, ^{

// 交换实例方法
Method originalMethod = class_getInstanceMethod([self class], @selector(viewWillAppear:));
Method swizzledMethod = class_getInstanceMethod([self class], @selector(lc_viewWillAppear:));

// 交换类方法时使用下面的:
// Method originalMethod = class_getClassMethod(object_getClass((id)self), originalSelector);
// Method swizzledMethod = class_getClassMethod(object_getClass((id)self), swizzledSelector);

BOOL didAddMethod = class_addMethod([self class],
@selector(viewWillAppear:),
method_getImplementation(swizzledMethod),
method_getTypeEncoding(swizzledMethod));

if (didAddMethod) {
class_replaceMethod([self class],
@selector(lc_viewWillAppear:),
method_getImplementation(originalMethod),
method_getTypeEncoding(originalMethod));
} else {
// 交换方法
method_exchangeImplementations(originalMethod, swizzledMethod);
}
});
}

#pragma mark - 实现自定义方法
- (void)lc_viewWillAppear:(BOOL)animated {
[self lc_viewWillAppear:animated];
NSLog(@"viewWillAppear: %@", self);
}

@end

# 交叉后的Runtime执行顺序：

  1. 根据@selector(viewWillAppear:) 找到自定义的 lc_viewWillAppear 实现；
  2. 自定义的 lc_viewWillAppear 实现，向 self 发送 @selector(lc_viewWillAppear) ；
  3. 根据 @selector(lc_viewWillAppear) ，找到默认的SDK中 viewWillAppear的实现；
  4. 执行默认SDK中 viewWillAppear 代码;
  5. 执行NSLog(@"viewWillAppear: %@", self);
  6. 结束；

# V. 动态添加方法

# 用途：

  • 如果一个类方法非常多，加载类到内存的时候也比较耗费资源，需要给每个方法生成映射表，可以使用动态给某个类添加方法解决。
  • 经典面试题：有没有使用 performSelector，其实主要想问你有没有动态添加过方法。

# 方法：

1.动态添加：

// 动态添加方法就不会报错
[p performSelector:@selector(eat)];

2. 重写 resolveInstanceMethod 方法：

  • 当一个对象调用未实现的方法，会调用这个方法处理,并且会把对应的方法列表传过来。
  • 刚好可以用来判断，未实现的方法是不是我们想要动态添加的方法
+ (BOOL)resolveInstanceMethod:(SEL)sel{

if (sel == @selector(eat)) {
// 动态添加eat方法
class_addMethod(self, @selector(eat), eat, "v@:");
// 第一个参数：给哪个类添加方法
// 第二个参数：添加方法的方法编号
// 第三个参数：添加方法的函数实现（函数地址）
// 第四个参数：函数的类型，(返回值+参数类型) v:void @:对象->self :表示SEL->_cmd
}
return [super resolveInstanceMethod: sel];
}
