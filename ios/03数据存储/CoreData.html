CoreData 的使用

简介:

  1. CoreData 是数据持久化存储的最佳方式.
  2. CoreData 是基于 sqlite 的封装, 数据保存到一个数据库文件中(表).
  3. iOS5 以后才出现.
  4. 它提供了对象-关系映射(ORM)的功能, 既能够将oc对象转化成数据保存在sqlite文件中, 也能将保存在数据库中的数据还原成oc对象.
  5. 数据操作期间, 我们不需要编写任何的sql语句.
  6. 数据最终的存储类型可以是：SQLite数据库，XML，二进制，内存里，或自定义数据类型.

好处：能够合理管理内存,避免使用sql的麻烦, 更高效. 不需要我们手动去创建model类, 可以通过一些操作, 制动生成.


# 一. 工作原理

  1. NSManagedObjectContext 临时数据库 向__NSPersistentStoreCoordinator__ 持久化存储助理 发送一个key(model名字)
  2. NSPersistentStoreCoordinator 通过这个key 在 NSManagedObjectModel 数据模型中 找到这个model 对应的 表
  3. NSManagedObjectModel 将这个表名 返回给 NSPersistentStoreCoordinator
  4. NSPersistentStoreCoordinator 通过 表名 找到 给表的 file 路径
  5. NSPersistentStoreCoordinator 将这个路径 返回给 NSManagedObjectContext
  6. NSManagedObjectContext 对 数据进行 处理(增, 删 , 该, 查)
  7. NSManagedObjectContext 保存数据 save

# 二. 使用步骤

# 创建模型文件:

一个模型文件相当于一个数据库.

# 添加实体:

一个实体相当于一个表.(点击下面add entity 添加表, 点击atttibutes的+号 添加属性)

# 创建实体类:

相当于模型类model.

# 生成上下文:

生成上下文,来关联模型文件生成数据库: 生成上下文时, 如果有多个数据库,且生成多个数据库文件时, 要使用不同的上下文进行操作.

# 从数据库查询对象:

过滤查询, 排序查询, 分组查询等等

# 更新数据:

前提: 先查询, 再修改数据

# 删除数据:

前提: 先查询, 再删除数据

# 多表关联:

在relationships添加与其他表的关联

注意: 添加完关联后, 该表的实体类要删除重新生成

# 多个数据库:

生成上下文时, 如果有多个数据库,且生成多个数据库文件时, 要使用不同的上下文进行操作.


# 三. 构成

(1)NSManagedObjectContext（被管理的数据上下文）

操作实际内容（操作持久层） 临时数据库 作用：插入数据，查询数据，删除数据

(2)NSManagedObjectModel（被管理的数据模型）

数据库所有表格或数据结构，包含各实体的定义信息 数据模型 作用：添加实体的属性，建立属性之间的关系 操作方法：视图编辑器，或代码

(3)NSPersistentStoreCoordinator（持久化存储助理）

相当于数据库的连接器 作用：设置数据存储的名字，位置，存储方式，和存储时机

(4)NSManagedObject（被管理的数据记录）

相当于数据库中的表格记录

(5)NSFetchRequest（获取数据的请求）

相当于查询语句 查询时 要建立请求对象

(6)NSEntityDescription（实体结构）

相当于表格结构

(7)后缀为.xcdatamodeld的包

里面是.xcdatamodel文件，用数据模型编辑器编辑 . 编译后为.momd或.mom文件

# 五. SQLite 和 CoreData 的对比

  1. sqlite是纯C语言存储方式.
  2. CoreData是对sqlite的封装, 它的数据的保存直接使用对象, 不用再写sql语句.
  3. 性能方面: sqlite 优于 CoreData
  4. 什么时候使用sqlite 和 CoreData? * 如果数据库表关联比较复杂时, 三四张表都关联, 那就使用sqlite * 如果数据库表结构比较简单, 表之间没有关系, 使用CoreData
